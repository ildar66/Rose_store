• Краткое описание.

              Удаление документа в списке документов.

  • Предусловия(pre-conditions).

              Запросить список документов.

  • Основной поток событий.
    1. Вариант использования начинается, когда загружены документы клиента.
    2. Клиент выбирает из списка документов нужную запись.
    3. Клиент выбирает из списка доступных действий - «Удалить».

       Если во время действия возникают ошибки выполняеться поток ошибок Е1.

    1. Система удаляет документ и удаляет его в базе данных.
    2. Вариант использования завершается.
  • Поток ошибок.

              Е1:ошибка при удалении документа :

1.        Система сообщает клиенту об ошибке и дает номер телефона службы поддержки банка клиента.

2.        Вариант использования завершаеться.

  • Альтернативный поток событий.
  • Постусловия(post-conditions).

              Обновленный список документов.
